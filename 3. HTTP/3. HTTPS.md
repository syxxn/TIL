# HTTPS

> Hyper Text Transfet Protocol Secure Socket Layer



### HTTPS 동작 원리

[인증서 발급] 

1. 인터넷 사이트는 자신의 정보와 공개키를 인증기관(CA)에 제출한다. 

2. 인증 기관은 제출된 데이터 검증절차를 거쳐 개인키로 사이트에서 제출한 정보를 인증기관의 공개키로 암호화하여 서버에 보낸다.

(SSL 인증서 : 인증기관의 이름, 서버의 공개키, 암호화 방법 등을 담음)

3. 인증기관은 웹 브라우저에게 자신의 공개키를 제공한다.

​    

[사이트에 접속시]

1. 사용자가 사이트에 접속하면 자신의 인증서를 웹 브라우저로 보낸다. 

2. 웹 브라우저는 미리 받았던 공개키로 인증서를 복호화(해독)하여 검증한다. 

3. 그러면 사이트의 정보와 공개키를 알 수 있게 된다. 사이트 공개키로 대칭키를 암호화해서 다시 사이트에 보낸다.

4. 사이트는 개인키로 암호문을 해독해 대칭키를 얻게 되고, 이제 대칭키로 데이터를 주고받을 수 있게 된다. 

5. 웹 브라우저와 사이트는 안전하게 전달된 대칭키로 통신을 하며, 세션이 종료되면 대칭키를 폐기시킨다.



>  기존 기능이 올바르게 작동하는지 확인함으로써 기능에 대한 불확실성을 감소시킬 수 있으며, 개발단기 초기에 문제를 발견할 수 있게끔 도와준다.
