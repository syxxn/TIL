# 마이크로프레임워크 vs 풀 스택 프레임워크

> 프레임워크 = [클래스 + 라이브러리]인 뼈대

## 마이크로프레임워크(micro-framework)

: 핵심적인 기능만을 가진 프레임워크

> API 서버에 최적화되어있는, 서버만 주로 처리하는 프레임 워크

+ 마이크로프레임워크는 최소한의 기능만 제공하여 개발자가 덧붙이는 식으로 개발한다

+ 대표적인 마이크로프레임워크로는 Python-flask, Ruby-Sinatra...가 있다

+ 마이크로프레임워크는 가능한 모든 서비스를 분할해서 관리하고, 필요할 때마다 서비스간의 통신으로 연결하여 거대한 서비스를 만든다

+ 기본적인 MVC model을 따르지 않으며, 다른 서비스나 애플리케이션을 위한 API의 빌드를 목적으로 설계된 경우가 많다



## 풀 스택 프레임워크(full-stack framework)

: 웹 개발에 필요한 요소를 모두 갖춘 웹 프레임워크



+ 대표적인 풀스택 프레임워크에는 Spring, Django, Ruby on Rails ...가 있다

+ 서버에서 뷰를 <u>렌더링</u>해서 서버-클라이언트를 다 처리할 수 있다

  > 렌더링 : 웹 표준에 맞도록, 화면에 적절히 그려내는 것

+ 웹 개발에 필요한 요소를 모두 갖추었다.

  > + 폴더 구조 및 명명 규칙
  > + 빌드 및 테스트 환경
  >
  > + 충실한 명령행 도구

+ 웹 프레임워크 뿐만 아니라 ORM이나 그런 부분도 번들(Bundle, 묶음)로 제공하는 경우도 있다

+ 풀스택 프레임워크는 다음과 같은 기능을 갖추고 있다

  >  모든 풀 스택이 이런 기능을 갖춘 것은 아니지만 맥락 상

  + request / response 추상화
  + 세션 관리, 사용자 인증 및 권한 권리
  + 웹페이지 템플릿 및 URL 매핑
  + 보안 기능
  + Data Model 관리
  + 기타 기능



____



규모가 작은(API가 적은) 서버를 구축할 때는 최소 기능만 가지고 간단하게 서비스를 제작하고자 마이크로 프레임워크를 사용한다

> 풀스택 프레임워크를 사용하면 여러가지 설정 때문에 초기비용이 더 든다

반대로 규모가 큰 서비스의 경우에는 초기 설정만 하면 많은 기능을 지원해주는 풀스택 프레임워크가 적합하다

> 오히려 서비스에 들어가는 모든 기능들을 설정하는 것이 더 많은 비용이 들기 때문이다
