# IP (Internet Protocol)

: 인터넷상에 있는 컴퓨터의 고유한 주소 / 네트워킹이 가능한 장비를 식별

-> OSI 네트워크 계층에서 호스트의 주소지정과 패킷 분할 및 조립 기능을 담당한다.



IP의 정보는 패킷 혹은 데이터그램이라고 하는 덩어리로 나뉘어 전송된다

IP에서는 이전에 통신한 적 없는 호스트에 패킷을 보낼 때 경로 설정이 필요 없다



IP는 **비신뢰성**과 **비연결성**이 특징이다

>  흐름에 관여하지 않기 때문에 보낸 정보가 제대로 갔는지 보장하지 않음

패킷 전송과 정확한 순서를 보장하려면 TCP 프로토콜과 같은 IP의 상위 프로토콜을 이용해야 한다



컴퓨터를 서로 구분하고 찾아가기 위해 사용하는 프로토콜이 **IP**이다.

> IP  : 인터넷상에 있는 컴퓨터의 고유한 주소로, 인터넷상의 한 컴퓨터에서 다른 컴퓨터로 데이터를 주고 받을 수 있다. 일반적으로 32비트로 나타내고 8비트씩 구분하여 표기한다. `123.123.123.123`(IPv4. 최대 12자리)

IP를 이용해 데이터를 보낼 때 **송신 컴퓨터의 위치**와 **수신 컴퓨터의 위치**를 적은 특정 형태 데이터를 만드는데, 이 데이터 형태를 **패킷**이라고 한다.

> 패킷 : 네트워크를 통해 전송하기 쉽도록 자른 **데이터 전송 단위**. 
>
> Packet = Header + Data +Trail(주로 에러 검출에 사용, 생략되는 경우도 多)

패킷 외에 메세지 형태로 보고하는 **ICMP**가 있다

> ICMP (인터넷 제어 메세지 프로토콜) 
>
> : TCP/IP에서 IP 패킷을 처리할 때 발생되는 문제를 알려주는 프로토콜
>
> -> 단독으로 통신하는 것이 아니라 IP프로토콜을 보조하는 것
>
> > 오류보고메세지 / 질의 메세지

### IP 주소

IP주소는 인터넷에서만 사용되는 전화번호라고 생각할 수 있다. 많은 전화번호를 일일이 외울수 없으므로 DNS를 이용하여 도메인 네임을 지정하는 것이다.



현재 인터넷에서 사용하는 표준 프로토콜은 IPv4(32bit)이다. 그러나 IPv4는 주소공간 고갈 문제를 겪고 있어 IPv6(128bit)가 도입되고 있다. 현재 널리 쓰이고 있는 대부분의 운영체제는 IPv6를 지원한다

> 127로 시작하는 IPv4 주소는 자기 자신을 가리킨다



IP 주소의 종류

+ 유동 IP

  : 웹상에서 접속할 때마다 IP의 주소 값이 바뀌는 것

  값이 자꾸 바뀌어 서버나 사이트를 외부에서 접근하기 쉽지 않음

  대부분의 개인 사용자가 이용

  

+ 고정 IP

  : 숫자가 고정되어 변경되지 않는 것

  나만 사용할 수 있어서 보안성이 좋지만 가격이 비쌈
