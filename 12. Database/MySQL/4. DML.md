# DML

(Data Manipulation Language) : 데이터 조작어



+ **SELECT** : 검색

  > SELECT : 검색하고자 하는 데이터(칼럼)를 나열
  >
  > DISTINCT : 중복행 제거
  >
  > ALIAS : 나타날 칼럼에 대한 별칭 부여
  >
  > FROM : 선택한 칼럼이 있는 테이블 명시

  기본 : `SELECT * FROM (테이블);`

  `SELECT (칼럼명,칼럼명) FROM (테이블); `

  특정 칼럼 검색 : `SELECT (칼럼1),(칼럼2),(칼럼3) FROM (테이블);`

  칼럼에 ALIAS 부여 : `SELECT (칼럼1) AS (별칭1),(칼럼2) AS (별칭2) FROM (테이블);`

  칼럼 합성(concatenation) : `SELECT CONCAT((칼럼1),'-',(칼럼2))AS '(별칭1)-(별칭2)'`

  > `SELECT CONCAT(EMPNO,'-'DEPTNO)AS '사번-부서번호' FROM EMPLOYEE;`
>
  > -> CONCAT은 문자열을 합쳐주는 함수

  중복행의 제거 : `SELECT DISTINCT (칼럼) FROM (테이블);`
  
  내림차순 정렬 : `SELECT (칼럼1),(칼럼2),(칼럼3) FROM (테이블명) ORDER BY (기준이 될 칼럼) DESC;`
  
  > `SELECT empno, name, job from employee order by name desc;`
  >
  > employee 테이블에서 직원의 사번(empno), 이름(name), 직업(job)에서 이름을 기준으로 내림차순 정렬한 것을 출력하시오.





+ **INSERT** : 등록/입력

  `INSERT INTO (테이블명(필드1,필드2,필드3...)) VALUES (필으1의 값, 필드2의 값, 필드3의 값...);`

  > + 디폴트 값이 세팅되는 필드 생략 가능
  > + 필드명을 생략했을 경우에는 모든 필드 값을 반드시 입력해야 함

  > `INSERT INTO ROLE(role_id, description) values(200,'CEO');`
  >
  > 롤 테이블에 role_id는 200, description에는 'CEO'로 한건의 데이터를 저장





+ **UPDATE** : 수정

  `UPDATE (테이블명) SET (필드1=필드1의값),(필드2=필드2의 값)... WHERE (조건식);`

  > `UPDATE ROLE SET description = 'CTO', WHERE role_id = 200;`
  >
  > ROLE테이블에 ROLE_ID가 200일 경우 DESCRIPTION을 'CTO'로 수정





+ **DELETE** : 삭제

  `DELETE FROM (테이블명) WHERE (조건식)`

  > `DELETE FROM ROLE WHERE role_id=200;`
  >
  > 롤 테이블에서 롤 아이디가 200인 정보를 삭제
