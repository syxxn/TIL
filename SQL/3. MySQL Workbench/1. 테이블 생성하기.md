### 테이블 생성하기

#### 1. SQL 문으로 생성하는 방법

#### 2. CSV 파일을 임포트(import)하는 방법

> CSV : Comma Separated Values : 모든 값이 콤마로 구분된 형식의 내용을 가진 파일



Workbench에는 CSV 파일을 그대로 테이블로 만들어주는 기능이 있다.



CSV 파일은 데이터 분석 분야에서 아주 흔하게 사용되는 파일의 형식이다. 그리고 현재 많은 회사들이 사용하는 엑셀에는 저장된 데이터를 CSV 파일로 추출할 수 있는 기능이 있는데, 이 말은 곧 **엑셀 파일 -> CSV 파일 -> 테이블**

이런 식의 변환을 통해 엑셀 파일로 존재하던 데이터를 SQL로 분석할 수 있다.

### Primary Key 설정하기

> 실무에서는 기본키가 설정된 데이터를 받는 경우가 더 많을 것임
>
> **꼭 모든 테이블에 PK가 필요한 것은 아니지만, 특별한 경우가 아닌 이상 존재하는 것이 바람직하다**



primary key(기본키)란 **테이블에서 하나의 row를 고유하게 식별할 수 있도록 해주는 column**을 말한다. 특정 컬럼을 PK로 설정하면 PK에 같은 값이 있는 row가 추가되는 것ㅇ르 DBMS가 자동으로 막아주기 때문에 중복된 row가 생길 위험성이 사라진다.



#### Primary Key의 종류

PK의 종류에는 크게 두가지가 있다

**1. Natural Key**

어떤 `개체가 갖고 있는 속성을 나타내는 컬럼`이 PK가 됐을 때, 이를 Natural Key라고 한다.

> 회원의 속성을 직접적으로 나타내는 PK

**2. Surrogate Key**

어떤 `개체의 실제 속성은 아니지만 Primary Key로 쓰기 위해 추가한 컬럼`을 Surrogate Key라고 한다. 이런 Surrogate Key에는 주로 1부터 순차적으로 증가하는 숫자가 들어가게 된다.



Natural Key는 그 값이 나중에 변경되면 모든 row의 값을 다시 수정해야 한다는 문제가 있기 때문에 보통은 Surrogate Key를 선택하는 경우가 더 많다.

### Not Null

Null은 값이 존재하지 않는 상태를 나타내기 위한 키워드이다. Null은 0과 다르다.

0은 0이라는 값이 있는 것이고, Null은 칸 자체가 비어있는 것

따라서 `Not Null`은 이 컬럼에는 반드시 값이 들어있어야 한다고 선언하는 것이다.



Primary Key는 특정 row를 식별하기 위한 칼럼으로써, Not Null이어야 한다.

### Auto Increment

> AI : Auto Increment '자동 증가'



id 컬럼에 항상 유일한 값이 들어간다는 보장만 있으면, 누군가가 대신 id 컬럼의 값을 넣어주는 게 더 편리할 것이다. Auto Increment 속성을 컬럼에 설정하면, 해당 컬럼에 관해서는 DBMS가 '자동으로 증가'하는 값을 넣어준다.



PK가 Surrogate Key인 경우에는 보통 Auto Increment 속성이 설정되어 있을 때가 많다.
