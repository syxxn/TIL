# SQL 작성 형식



### 1. SQL 문 끝에는 항상 세미콜론을 써 줘야 한다

 SQL 문법 상 세미콜론이 하나의 SQL 문을 종결하는 단위이기 때문에, 하나의 SQL 문 끝에는 세미콜론을 반드시 써 줘야 한다



Workbench에서는 SQL문이 하나인 경우, 세미콜론을 입력하지 않아도 알아서 인식하고 실행해 주지만, 여러 개의 SQL문이 있을 때는 세미콜론이 없으면 실행되지 않는다.





### 2. SQL문 안에는 공백이나 개행 등을 자유롭게 넣을 수 있다

SQL문은 세미콜론으로 문장이 끝난 것을 확인하기 때문에, 공백이 있거나, 개행을 해도 문제가 없다.

이런 점을 이용해서 길이가 긴 SQL 문을 작성할 때는 개행, 탭 등을 적절하게 활용해서 가독성을 높이는 것이 좋다.



### 3. SQL문의 대소문자 구분 문자

MySQL의 예약어는 대문자로 적는 것이 관례이고, 보기에도 좋다.

데이터베이스 이름, 테이블 이름, 컬럼 이름 등은 대소문자를 가리지 않고, 실제로 존재하는 것의 이름을 적어주면 된다.



SQL문은 대소문자를 구분하지 않지만, 가독성을 위해 '예약어' 만큼은 꼭 대문자로 쓰는 습관을 들이자



### 4. 데이터베이스 이름과 테이블 이름

`SELECT * FROM copang_main.member;`

데이터베이스 이름 뒤에 점을 붙이고 그 다음에 테이블 이름을 쓴 것이다. 실무에서는 서로 다른 데이터베이스에 같은 이름의 테이블이 존재할 수도 있기 때문에, 명확하게 하기 위해서 위 처럼 쓰는 것이 좋다.



`USE (데이터베이스);`를 이용하여 어떤 데이터베이스를 쓸 것인지 확실하게 정해두면,

`SELECT * FROM member;`처럼 테이블 이름만 써도 괜찮다.
