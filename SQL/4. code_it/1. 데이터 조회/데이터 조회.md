### ORDER BY

> 명령문의 제일 마지막에 나와야 한다

+ 키를 기준으로 정렬하기

   + 오름차순 `SELECT * FROM copang.main ORDER BY height ASC;`

   + ​                `SELECT * FROM copang.main ORDER BY height;` 

     > 하나만 있을 때는 ASC를 생략해줘도 되지만, 요청이 여러개일 때는 반드시 표기해야 한다.

  	+ 내림차순 ` SELECT * FROM copang.member ORDER BY height DESC;` 

+ 키는 내림차순으로 몸무게는 오름차순으로 정렬하기

  `SELECT * FROM copang.member ORDER BY height DESC, weight ASC;`

  > 먼저 선언한 컬럼을 우선으로 해서 정렬이 차례대로 수행된다



### CAST

숫자값이 담긴 컬럼을 정렬 기준으로 할 때는 그 컬럼의 데이터 타입이 숫자형인지, 문자열형인지를 잘 살펴봐야 한다. 이미 TEXT 타입인 컬럼에 있는 숫자값들을, 그냥 INT 등의 숫자형 타입으로 보고 정렬할 수도 있다.

정렬할 때 그 컬럼의 값들의 데이터 타입을 일시적으로 변경해주면 된다. 이걸 하기 위해서는 `CAST`라는 함수를 써야한다. CAST는 '틀을 만들다','틀을 정하다'라는 뜻을 가진 단어로, 보통 프로그래밍 세계에서 어떤 변수의 데이터 타입을 바꿀 때 사용된다. 



`SELECT * FROM FOR_TEST.ordering_test ORDER BY CASE(data AS signed) ASC;`

위의 표현식은 data 컬럼에 존재하는 값들의 데이터 타입을 일시적으로 signed라는 데이터 타입으로 변환하라는 뜻이다. signed 양수와 음수를 포함한 모든 정수를 나타낼 수 있는 데이터 타입이다. 만약 문자열 타입으로 저장된 숫자값에 소수점이 포함되어 있다면 signed 대신 decimal(소수점이 있는 수를 나타내는 타입)을 사용하면 된다.
