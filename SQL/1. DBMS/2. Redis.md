# Redis

> REmote DIctionary Server

Redis는 NoSQL의 종류 중 하나로서 메모리 기반의 "키-값" 구조 데이터 관리 시스템이다.

모든 데이터를 메모리에 저장하고 조회하기에 빠른 Read, Write 속도를 보장하는 오픈소스 기반의 **비관계형 데이터베이스**이다.



모든 데이터를 메모리로 불러와서 처리하는 메모리 기반 DBMS이기 때문에 읽기와 쓰기 명령이 매우 빠르다. 데이터를 지속적으로 유지하기위해 모든 작업을 로그에 기록해서 디스크에 저장한 후, 시스템을 구동할 때 로그를 기반으로 데이터를 다시 메모리에 올리는 방식을 사용한다.

레디스는 **램**이다. 램은 휘발성 메모리이므로, 레디스는 데이터를 영구적으로 보존할 수 없다.

따라서 전체 데이터를 영구히 저장하기 보다는, 캐시처럼 휘발성이나 임시성 데이터를 저장하는 데 많이 사용된다.



### Redis의 특징

+ **Key/Value**

  redis는 Map 데이터 저장소로서 Key값과 value값으로 이루어져 있다.

  따라서 key 값을 통해 value 값에 쉽게 접근하고 쉽게 저장이 가능하다.

+ **다양한 데이터 타입**

  + String : 가장 다양한 데이터 타입.

  + Lists : 삽입 순서에 따라 정렬된 문자열의 집합. 기본적인 linked list 형태
  + Sets : 고유하고 정렬되지 않은 문자열의 집합
  + Sorted Sets : Sets와 비슷하지만 모든 값은 score(내림차순)로 정렬됨
  + Hashes : value와 연관된 필드로 구성된 맵으로 모든 필드와 값은 문자열로 구성된다.

+ **In-Memory data structure store**(저장소)

  redis는 디스크가 아닌 메모리 기반의 데이터 저장소이다.

+ 데이터가 메모리 + Disk에 저장된다

  > 프로세스가 죽거나 장비가 Shutdown되더라도 Data 복구가 가능하다

+ 명시적으로 삭제, Expire(만기)를 설정하지 않으면 데이터는 삭제되지 않음(=영구적 보존)

  > 하드디스크에도 저장하기 때문



### Redis의 사용사례

+ 캐싱

  : 위에서도 언급했듯이 `In-Memory data structure store` 이기 때문에 캐시의 역할을 수행할 수 있다.

  **다른 데이터베이스의 부하를 막아주는 역할**을 합니다.

+  세션 관리

  유저의 정보 관리 등 세션으로서 redis 를 많이 사용하고 있다.

+ 실시간 순위표

  redis는 sorted set이라는 데이터 타입이 존재하는데, 이를 통해 원하는 값으로 자동 정렬되어 순위표를 구성할 수 있다.

+ 대기열

  redis는 list라는 데이터 타입이 존재하는데, 이를 통해 대기열을 손쉽게 구현할 수 있다.

+ 채팅

  실제 채팅 서비스에서도 많이 사용되고 있다.

  기본적으로 PUB/SUB 표준을 지원하기 때문에, 고성능 채팅방, 서버 상호 통신 등을 구현할 수 있다.
