# 인덱싱(Indexing)



### 인덱스

> 인덱스는 데이터베이스 분야에 있어서 테이블에 대한 동작의 속도를 높여주는 자료구조를 일컫는다. 

테이블에 저장된 데이터를 빠르게 조회하기 위한 데이터베이스 객체를 말한다. 인덱스(Indes, 색인)는 말 그대로 책의 맨 처음 또는 맨 마지막에 있는 색인(`찾아보기`)이라고 할 수 있다. 이 비유를 가져와서 인덱스를 살펴본다면 데이터는 책의 내용이고, 데이터가 저장된 레코드의 주소는 인덱스 목록에 있는 페이지 번호가 될 것이다.



DBMS의 인덱스는 항상 정렬된 상태를 유지하기 때문에 원하는 값을 탐색하는데는 빠르지만, 새로운 값을 추가하거나 삭제, 수정하는 경우에는 쿼리문 실행 속도가 느려진다. 결론적으로 **DBMS에서 인덱스는 데이터의 저장 성능을 희생하고 그 대신 데이터의 읽기 속도를 높이는 기능**이다. `SELECT` 쿼리 문장의 `WHERE` 조건절에 사용되는 칼럼이라고 전부 인덱스로 생성하면 데이터 저장 성능이 떨어지고 인덱스의 크기가 비대해져서 역효과를 불러올 수 있다.



### 인덱싱

인덱싱은 데이터 검색에서 발생하는 비효율적인 문제를 해결하기 위한 대표적인 방법이다. 즉 **DBMS에서 요청된 레코드에 빠르게 접근할 수 있도록 데이터와 관련된 부가적인 구조인 인덱스(index, 색인)를 관리**하는 것이다. 그리고 인덱스를 디자인하고 생성하는 작업(테이블에 인덱스를 삽입)을 인덱싱이라고 한다.



인덱스의 평가기준

1. 접근 시간 : 사용자가 원하는 데이터를 찾는 데 걸리는 시간
2. 유지비용 : 새로운 데이터 삽입 및 기존 데이터 삭제 연산으로 인한 인덱스 구조 갱신 비용
3. 공간 비용 : 인덱스 구조에 의해 사용되는 부가적인 공간 비용



+ 기존의 테이블에 인덱스 추가하기

`ALTER TABLE 테이블명 ADD INDEX (필드명(크기));`

+ 테이블 생성시 인덱스 추가하기

`CREATE TABLE 테이블명 (필드명 데이터타입(데이터크기),INDEX(필드명(크기))) ENGINE MyISAM;`

> MyISAM : MySQL의 기본 저장 엔진 타입
