# IoC DI



### Inversion of control

호출을 다른데로 맡기는 **제어권 역전**

코드 부품을 짜 놓고 실행은 프레임워크한테 맡기는 것을 IoC라고 한다.



개발자는 개발해! 개발 안하고 일일이 실행하고 있으면 시간낭비야 그거!

```java
public class A {

    @Autowired //빈 등록
    private B b;

}
```





### Dependency Injection

의존성(저거 없으면 이거 안 돌아가) 주입

개발자가 코드를 직접적으로 개발하는 게 아니라, 추상적인 개념(interface)을 만들고, 그것을 구현해서 개발한다.

DI를 쓰게 되면, 모듈 간의 결합도가 낮아지고 유연성이 높아진다.

> 결합도가 낮아짐으로써 한 클래스를 수정하였을 때, 다른 클래스도 수정해야 하는 상황을 막을 수 있다.

스프링에서는 IoC Container에 다 들어가 있다. 인터페이스든, 클래스든 ... (빈에 등록됨)
