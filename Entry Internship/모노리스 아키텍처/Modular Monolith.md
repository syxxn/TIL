# Modular Monolith

: 모듈형 모노리스는 상호 교환 가능한(그리고 잠재적으로 재사용 가능한) 모듈에 중점을 두고 모노리스를 설계하는 접근 방식이다.

> 자바에서는 패키지로 모듈을 나눈다

> 모노리스는 정확히 **하나의 배포 단위**가 있는 시스템이다

**서비스마다 나눈 뒤 하나의 애플리케이션으로 묶어주는 것**을 모듈형 모노리스라고 한다. 



모듈(module)

> 프로그램 내부를 기능별 단위로 분할한 부분/ 교환이 쉽도록 설계되어 있을 때의 그 각 구성 요소

모듈화(Modularity)

> 소프트웨어 시스템을 분해하여 서브시스템들과 컴포넌트들의 그룹으로 묶는 것
>
> 주요 목적은 시스템 내에 잘 정의한 경계를 도입해 소프트웨어 복잡성을 다루는 것

모듈형

> 결합 될 때 완전한 전체를 형성하는 별도의 부품으로 구성하고, 
>
> 함께 결합하여 더 큰 물체를 형성할 수 있는 별도의 부품 세트로 구성



모듈은 **독립적이고 상호 교환 가능해야 한다**. 물론 완전히 독립적인 것은 불가능하다. 다른 모듈과 통합되지 않는다는 의미이기 때문이다. 모듈은 항상 어떤 것에 의존하지만, 의존성은 최소한으로 유지해야 한다. 

**모듈에는 원하는 기능을 제공하는 데 필요한 모든 것이 있어야 한다.**



### 장점

#### 재사용성 향상

모듈식 구성 요소는 재사용 및 용도 변경이 가능하며 팀이 단일 정보 소스를 설정하는 데 도움이 된다. 이것은 더 빠르고 일관된 개발로 이어질 수 있다.

#### 더 잘 구성된 종속성

모듈형 모노리스는 개별 기능을 더 독립적으로 만들기 때문에 본질적으로 종속성을 더 체계적이고 가시적으로 만든다. 이를 통해 개발자는 애플리케이션의 어떤 부분에 어떤 종속성이 필요한 지 쉽게 평가할 수 있다.

#### 증가된 관찰 가능성

모듈화를 염두에 두고 개발하면 개발자가 코드에 더 쉽게 액세스 할 수 있다. 특히 java 개발자의 경우 인터페이스 클래스와 필수 java-doc 주석은 모듈화 된 기능으로 작업해야 하는 개발자의 접근성과 상호 운용성을 향상시키는 데 도움이 된다.



### MSA vs Monolithic Architecture

#### MSA

> 마이크로서비스로 올바르게 시스템을 구축하기 위해서는 분산 시스템이 가진 복잡성을 해결해야 하며, 무엇보다 개별 마이크로서비스가 **독립성을 가지고 성장할 수 있는 아키텍처와 설계가 뒷받침** 되어야 한다.

+ 유연하게 결합된 서비스로 구성된다. 각 서비스는 단일 비즈니스 작업을 지원한다.
+ 클라이언트는 다른 애플리케이션일 수도 있다.
+ 각 마이크로 서비스는 마이크로 서비스가 해결하도록 설계된 문제에 가장 잘 맞는 프로그래밍 언어 및 프레임워크를 사용하여 개발할 수 있다.
+ 각 마이크로 서비스는 자체 데이터베이스를 사용할 수 있다(분산화)
+ 응용 프로그램의 다른 마이크로 서비스에 영향을 주지 않고 독립적으로 배치된다.
+ 단순하고 집중하며 독립적입니다. 따라서 응용 프로그램을 쉽게 유지 관리할 수 있습니다.
+ 각 마이크로 서비스는 다른 서비스와 독립적으로 확장할 수 있다.
+ 애플리케이션 기능이 여러 서비스에 걸쳐 분산되기 때문에 하나의 서비스가 실패한 경우 다른 서비스에서 제공하는 기능을 계속 사용할 수 있습니다.

#### Monolithic Architecture

+ 전체 애플리케이션은 단일 단위로 설계되고 개발 및 배포된다.
+ 애플리케이션 간 기능 재사용 기회가 제한된다.
+ 일반적으로 전체 응용 프로그램은 단일 프로그래밍 언어로 작성된다.
+ 전체 응용 프로그램에서는 하나 이상의 데이터베이스를 사용합니다(중앙 집중식)

+ 변경하면 전체 애플리케이션을 재배포하고 재시작해야 한다.
+ 애플리케이션 범위가 증가하면 코드가 더 복잡해진다.
+ 비즈니스 요구사항이 애플리케이션의 특정 부분만 스케일링하기 위한 경우에도 전체 애플리케이션의 스케일을 조정해야 한다.
+ 구성 요소의 오류는 전체 응용 프로그램의 가용성에 영향을 줄 수 있다.





일반적으로는 MSA가 아니면 Monolithic으로 개발하고 있다고 생각하면 된다. 

modular Monolith를 사용하면 MSA로 전환이 쉽다.
