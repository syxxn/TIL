# Software Architecture



Software Architect는 말 그대로 소프트웨어의 구조(아키텍처)를 설계하고 구현하고 보수하는 건축가이다. 견고해서 쉽게 무너지지 않고, 처리량이 늘어나거나 다른 소프트웨어를 추가해야할 경우 쉽게 대응이 가능한 유연한 구조를 만드는 것이 중요하다.



### 아키텍처 (구조)

집을 지을 재료들을 이용하여 설계대로 건축하면 단단하고 예쁜 집이 된다. 하지만 설계 없이 집을 막 지으면 금방 허물어져버린다. 따라서 미리 설계하고 개발하는 것이 중요하다.



큰 규모의 소프트웨어를 개발하게 된다면,

+ 복잡하다
+ 여러 요구사항, 변수, 외부 요인 등에 대처하기 어려워진다
+ 유지보수가 어렵다



### DDD

Domain Driven Design

DDD(도메인 주도 설계) : 비즈니스 도메인(서비스) 별로 나누어 설계하는 방식



온라인 서점을 만든다고 예를 들면, 온라인 서점도 하나의 도메인이며, 그 하위 도메인으로 도서, 주문, 배송 도메인이 나온다. 하위 도메인 별로 모듈을 나누어 개발하는 것을 DDD라고 한다.



**느슨한 결합과 높은 응집도로 독립적인 모듈을 개발**하는 것이 중요하다

> 연관성이 높을수록 응집도가 높다
>
> 결합은 다른 모듈끼리, 응집도는 애플리케이션 내부에서.
>
> 결합도를 낮추기 위해서는 공개된 인터페이스를 통해 통신하면 된다



이는 **복잡도를 감소**시키고, **효과적인 유지보수**, **독립성을 확보**한다.



DDD는 개발 방법론 중 하나인데, DDD로 개발하면 이런저런 아키텍처를 띈다.

+ Domain
  + user
    + controller
    + service
  + auth
    + controller
    + service
+ Global
