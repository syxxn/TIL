### 온프레미스

: 데이터 센터나 서버실에 서버를 두고 직접 관리하는 방식

+ 서버, 네트워크 장비, OS, 스토리지, 각종 솔루션을 직접 사서 설치하고 관리해야 함

  -> 직접 관리한다는 장점

  -> 고가의 장비를 사용하여 초기 투자 비용이 크다는 단점

  -> 사용 예측량을 가늠하기가 힘들며 한번 구축해놓으면 사용량이 적어도 유지 비용은 그대로 나간다는 단점

+ 서버가 죽지 않는 것을 목표로 함

### 퍼블릭 클라우드

: 인터넷을 통해 불특정 다수에게 서비스 형태로 제공되는 시스템

> AWS(Amazon Web Service), Microsoft Azure, GCP(Google Cloud Platform) 등 

+ 클라우드 프로바이더(제공자)가 데이터 센터와 인프라를 보유하고 있음

+ 서비스 형태라는 건 사용자가 원하는 옵션을 선택하고 사용한만큼 비용을 지불하면 된다는 것

+ 제공하는 서비스에 따라 laaS, PaaS, SaaS 등으로 나뉨

  > 1. LaaS : 클라우드를 통하여 저장장치 또는 컴퓨팅 파워를 인터넷을 통한 서비스 형태로 제공
  >
  >    기본적인 아키텍처
  >
  > 2. PaaS : 사용자가 소프트웨어를 개발할 수 있는 기반이 되는 서비스
  >
  >    LaaS 서비스 기반 또는 독자적인 아키텍처 모습
  >
  > 3. SaaS : 클라우드 서비스 사업자가 인터넷을 통해 소프트웨어를 제공하고 사용자가 인터넷상에서 이에 원격 접속해 해당 소프트웨어를 활용하는 서비스
  >
  >    클라우드의 최상위 계층
  >
  >    PaaS 서비스 기반 또는 독자적인 아키텍처

+ 많은 인스턴스로 이루어진 분산 환경에서 인스턴스가 죽으면 다른 인스턴스가 빠르게 대체하는 것



### 프라이빗 클라우드

: 퍼블릭 클라우드에서 이용자를 한정한 형태로, 보안이 좋고 독자적인 기능이나 서비스를 추가하기 쉽다

> ex) 기업 내 서비스



클라우드 서비스는 하드웨어를 빌리는 것!! 서버는 관리 해야 함

### 서버리스 컴퓨팅

: 클라우드 컴퓨팅 모델 중 하나로, 클라우드 제공자는 동적으로 머신 자원의 할당을 관리

> 가격은 애플리케이션이 소비한 자원의 실제 양에 기반을 둠

+ IT 인프라를 데이터 센터 혹은 클라우드에 별도 준비 없이, 필요한 기능을 함수 형태로 구현

  > 코드(다른 컴퓨팅 방법과 코드는 동일)만 올리면 함수를 실행 시켜줌
  >
  > 다른것과의 차이는 "배포"를 어떻게 하냐

+ 자동 스케일링 방식으로 시시각각 변하는 자원 수요를 지원하며 전통적인 백엔드 대신 사용한다.

+ **서버를 고려하지 않고** 애플리케이션과 서비스를 구축하고 실행할 수 있다

  > 백엔드인데 내가 관리하지 않는 것



#### 장점

1. 서버 관리 불필요
2. 유연한 규모 조정
3. 종량제 요금
4. 자동화된 고가용성



#### 단점

1. cold start

   > 자고 있는 함수를 깨우는데 시간이 걸림 (보통 서버는 항시 대기)

2. 서버 제공자에 대한 의존도 높음

   > 한 서버리스에서 다른 쪽으로 마이그레이팅하는 것은 쉽지 않음

   





트래픽의 변동이 많은 시스템은 **클라우드 시스템**이 유리하고, 

잠시라도 끊어져서는 안되는 시스템이나 클라우드 업체가 보장하는 것 이상의 가용성이 필요한 시스템에서는 **온프레미스**가 유리하다. 

각자 장단점이 있기 때문에 온프레미스와 클라우드를 함께 사용하기도 한다.

**서버리스 컴퓨팅**은 다양한 이벤트를 함수의 형태로 처리하는 특징을 가지고 있기 때문에 IT 서비스 운영시 예상되는 반복적인 일괄(batch) 처리 형태의 기능에 알맞다
