# Docker Swarm

: 도커가 공식적으로 만든 오케스트레이션 툴(**여러 호스트 서버의 컨테이너들을 배포 및 관리를 위한 툴**)이다.

> 쿠버네티스(구글) 등 오케스트레이션 툴은 많이 있다 

Docker Swarm을 이용하면 여러 개의 서버와 컨테이너 관리를 쉽게 할 수 있다.



Docker는 하나의 호스트를 기준으로 한다. 그러나 실제로 서비스 운영환경에서 도커를 운영한다면, 하나의 호스트 머신으로는 확장성 측면에서 힘들다. 그렇기 때문에 여러 대의 서버를 클러스터로 만들어 자원을 병렬적으로 확장하는 것이다.



오케스트레이션 툴은 컨테이너 배포뿐만 아니라 다양한 기능을 포함한다.

+ **스케줄링** 

  + 컨테이너 자동 배치 및 복제
  + 컨테이너으 수를 여러개로 나눠서 배포하고 컨테이너가 죽으면 즉시 컨테이너를 다른 서버에 배포하여 서비스에 문제가 없도록 함

+ **클러스터링**

  + 여러개의 서버를 하나의 서버처럼 묶어서 사용할 수 있도록 하는 기술

+ 클러스터 외부에 서비스 노출

  > cluster : 물리적으로 함께 저장되는 하나 이상의 테이블의 그룹. 
  >
  > 이 테이블은 공통 열을 공유하고, 종종 함께 사용되기 때문에 함께 저장된다.

+ 컨테이너 추가 또는 제거를 이용한 확장 및 축소

+ 컨테이너 서비스간의 인터페이스를 통한 연결 및 네트워크 포트 노출 제어

+ 컨테이너 그룹에 대한 로드밸런싱(부하분산)

+ 컨테이너 장애 복구



#### Docker Swarm을 쓰는 이유

1. 다른 툴을 설치할 필요가 없다
2. 도커 명령어를 그대로 사용할 수 있다
3. 쉽고 편리하다
4. 안정성이 검증되어 있다
