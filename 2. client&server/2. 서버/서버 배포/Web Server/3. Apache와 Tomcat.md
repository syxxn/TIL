# Apache

> 아파치 HTTP 서버 : 정적 컨텐츠 반환

: 1995년 처음 발표된 W3 서버용 소프트웨어로 세계에서 가장 많이 쓰는 HTTP 서버 이다.

##### 정적인 데이터를 처리하는 웹 서버



+ 오픈소스 라이선스에 따라 무료로 배포되며, 리눅스와 윈도우 등을 비롯해 거의 모든 운영체제와 시스템에서 운용이 가능하다

+ Apache는 다양한 추가기능에, 구축이 쉬우며, 오픈소스이기 때문에 많이 사용한다.

  하지만 Apache 자체만으로 엄청 무겁고, 취약점이 발견되었기에, 대형사이트 운영자나 프로그래밍이 능숙한 사람들은 NGINX, IIS를 주로 사용하고 있다.

+ 아파치는 기본적으로 80포트로 연결되며, Apache_HOME/bin/httpd.bat 같은 배치파일을 실행하거나, 서비스로 등록하여 서비스로 실행시키면 된다



### 톰캣(Tomcat)

> was : 동적 컨텐츠를 반환

: 어플리케이션 서버로서, <u>자바 서블릿</u>을 실시키고 JSP코드가 포함되어 있는 웹 페이지를 만들어준다.

> Java Servlet : 자바를 사용하여 웹페이지를 동적으로 생성하는 서버측 프로그램 혹은 그 사양, 흔히 "서블릿"이라 함.

+ 게시판처럼 사용자가 입력하고 거기에 따라 다른 결과를 보여주는 동적인 페이지에 대해서는 톰캣이 수행하는 것이 적합하다
+ **아파치와 톰캣을 연동한다는 것은 동일한 포트로 운영한다**는 의미로, 보통 이미지나 HTML 요소는 아파치에서 처리가 되고, 사용자의 데이터 처리와 같은 일은 톰캣에서 처리하게 한다.

+ 톰캣은 기본적으로 8080포트로 연결되며, 서버를 띄우려면 Tocat_HOME/bin/start.bat과 같은 배치파일을 실행시키면 된다
